# Завдання 8. Вимкнений DEBUG і повторна перевірка admin

1. У `base/settings.py` змінив `DEBUG = False`, залишивши дозволені хости `["127.0.0.1", "localhost"]`.
2. Запустив `python manage.py runserver` і відкрив `http://127.0.0.1:8000/admin/`.
3. Тепер замість детального traceback бачу лаконічну сторінку помилки (бо DEBUG вимкнено), а в консолі сервер знову нагадує про “You have 18 unapplied migration(s)”.

> Отже, коли DEBUG вимкнено, Django поводиться так, як на бойовому сервері: приховує технічні деталі, але проблема з міграціями нікуди не зникає.
